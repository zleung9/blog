# Publishing to Github

### Setting up your github page

1. There are two types of GitHub Pages: *Project Pages* and *User Pages*. Here we discuss how to publish site as a **Project Page**.

2. To publish a Pelican site as a Project Page you need to push the content of the `output` dir generated by Pelican to a repository’s `gh-pages` branch on GitHub.

3. `ghp-import`
> \$ pelican content -o output -s pelicanconf.py  
> \$ ghp-import output -b gh-pages  
> \$ git push origin gh-pages

The `ghp-import output` command updates the local `gh-pages` branch with the content of the `output` directory (creating the branch if it doesn’t already exist). The `git push origin gh-pages` command updates the remote `gh-pages` branch, effectively publishing the Pelican site.
